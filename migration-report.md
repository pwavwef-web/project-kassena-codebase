## 7. Known issues / limitations

- **Mobile native folders:** `android/` and `ios/` contain README placeholders
  only. Run `flutter create . --platforms=android,ios` (non-destructive) to
  generate real scaffolding before building APK/IPA.
- **Firebase hosting targets** (`web`, `studio`) are declared but **not mapped**
  to Hosting sites — `firebase deploy --only hosting` will fail until the owner
  runs `firebase target:apply` (see §8).
- **Indigen World parent-brand logo** does not exist yet (only the Project
  Kasena mark). OG image PNG export still pending.
- **Asset licensing** is undocumented per-file (see `asset-manifest.md`).
- **Cloud Functions** were preserved as-is; they target existing collections and
  should be reviewed before reuse in new flows.
- `npm audit` reports 1 low-severity advisory in the new dev dependencies.

## 8. Firebase — owner actions required (none performed by migration)

1. Confirm/keep the project id in `.firebaserc` (`project-kassena-7e026`).
2. Create Hosting sites and map targets:
   ```bash
   firebase target:apply hosting web    <web-site-id>
   firebase target:apply hosting studio <studio-site-id>
   ```
3. Build apps (`npm run build:web && npm run build:studio`) before any deploy.
4. Cloud Functions secrets (Paystack secret key, SMTP) belong in Functions
   config / Secret Manager — never in client apps.
5. App Check and auth-provider changes are **deliberately untouched**; decide on
   these separately.
